Main Screen

The main screen is where you can quickly view your reminders and by using a combination of the Reminder States, Indicators, Filter options, and Grouping options quickly see the information you are interested in.

 

Reminder Status/States

          At any given time a reminder will always be in 1 of 3 states:

Active

Active reminders are reminders which are ready to be completed or performed. For example you have a reminder created to take out the trash every Tuesday morning at 7:00AM. When Tuesday morning at 7:00AM arrives the reminder becomes Active and reminds you to complete this action.

Reminders can become Active via a couple of different methods:

·        The trigger condition associated with the reminder has been met

a.     For Date/Time reminders the date and time has occurred

b.     For location based reminders you have entered within the configured radius for the location

c.      For weather based reminders the weather condition set has occurred or is scheduled to occur

·        A snoozed reminder has met its snooze duration

·        A reminder that has been created without a trigger configured/enabled

·        A Scheduled reminder has been manually Activated

Active reminders will be removed from the Active state via any of the following actions;

·        Mark the Reminder as Done

·        Disable the Reminder

·        Delete the Reminder

·        Snooze the Reminder

·        Edit the reminder to where its trigger condition is no longer met

 

The following Actions can be taken on a reminder in an Active State; Done, Snooze, Edit, Disable, Delete

 

Scheduled

Reminders are Scheduled if:

·        The reminder has not yet reached its trigger criteria

o   For Date/Time reminders the Date and time has yet to occur

o   For location reminders the device has not been within the configured location

o   For weather based reminders the configured weather condition has not occurred

·        An active reminder configured to repeat has been marked Done

·        An active reminder has been Snoozed but has yet to reach its snooze duration

Scheduled reminders will be removed from the Scheduled state via any of the following actions;

·        The Trigger Condition is met

·        Activate the Reminder

·        Disable the Reminder

·        Delete the Reminder

·        Edit the reminder to where its trigger condition has been met

·        Edit the reminder to where it no longer has a trigger condition

The following Actions can be taken on a reminder in a Scheduled State; Activate, Edit, Disable, Delete

 

Inactive

Inactive reminders are those items that have been disabled or have been completed. In their current state they will not Alert you in the future but can be easily enabled or edited to be reused.

Inactive reminders are:

·        Reminders that have been manually Disabled

·        Non-Repeating reminders that have been marked Done

Inactive reminders can be removed from this state by:

·        Enabling a disabled reminder

·        Deleting a reminder

·        Edit the reminder to repeat

·        Edit the reminder and adjust the trigger criteria to a date in the future

 

The following Actions can be taken on a reminder in an Inactive State; Enable, Edit, Delete

 

Reminder Indicators

            Color - Active Reminders have a color coding to indicate how far past their due date they currently are.

Red  Have been Active for over 7 Days

 

Orange  Have been Active from 4 to 6 Days

 

Yellow  Are now 1 to 3 Days past the Due Date

 

Blue  Have become Active that Day.

          Icons  Reminders have different icons associated with then to provide quick information about that reminder at a glance.

    -           Indicates a reminder that was flagged as Important.

    -           Indicated that this reminder is dependent upon another reminder to complete before it can be scheduled.

    -           Reminders with this icon have been snoozed for an amount of time after already becoming active.

    -           Reminders that have a Date/Time trigger configured

    -           Reminders that have a Weather trigger configured

    -           Reminders that have a Location trigger configured

    -           Reminders that have at least 1 Step configured

    -           Reminders that have the email notification enabled

    -           Reminders that have local notification enabled

 

 

Reminder Filters

            Available at the bottom of the Main Screen are 3 different methods to filter your reminders and show only the information you are interested in. These can be used individually or in any combination you like.

If you have a filter applied when the app is launched these will flash to notify you that filters are applied and you may not see all reminders.

            Date  This filter allows you to show reminders with due dates that are scheduled to occur in the provided range (Today, This Week, etc.). Given the nature of this option this mostly only applies to reminders that have Date/Time triggers associated with them

            Group  Show only the group or groups you are interested in seeing at that time.

            State (Change to Attribute Filter)  Show only reminders in a given state (Active, Scheduled, Inactive) or those reminders flagged as Important

 

Reminder Actions

·        DONE  Marking an Active reminder done clears that from your active section. This should be done when you have completed the reminder. Reminders marked Done will become Inactive if no repeat options have been set, or become scheduled if repeat options have been configured.

·        UNDO  When marking a reminder done the item will be in a crossed out state that users can quickly undo to restore to a normal Active state. Once the screen is refreshed these previously active reminders are processed and this option is no longer available.

·        SNOOZE  Quickly defer your reminder to a time in the future. The Snooze options are:

o   In an Hour

o   In the Morning

o   Tonight

o   In 24 Hours

o   In a Week

o   In the Future

·        EDIT  Access the screen to update any and all details about this reminder.

·        DISABLE  Marks this reminder Inactive and do not notify me of this.

·        DELETE  Purges the reminder forever from your freeminders account. There is no recovery method for this reminder once deleted.

·        ACTIVATE  Manually make a scheduled reminder Active. The date/time you Activate this reminder is used to track due date going forward.

·        ENABLE  This will remove reminders from the inactive state. Enabling these reminders will either place them in the Scheduled or Active section depending on their trigger criteria.

·        REFRESH  Pulling down from the main screen will force a synchronization with the freeminders servers and process any items marked done.

 

 

Reminders

Create / Edit

To CREATE a new reminder press the + in the top right of the Main Screen.

To EDIT, locate the existing reminder you would like to edit on the Main Screen and slide it to the left. Press the EDIT option from the choices now shown.

When creating or editing a reminder there are a number of fields available for you to use. The intention of those is listed below.

·        Reminder Name  This is the only required field. To create a quick active reminder just supply a Name and save. These will show up in the Active section using the currently date and time to track the due date. (Examples; Pick up kids from school, Grab an umbrella, Your auto Insurance payment is due)

·        Reminder Group  Select a group for this reminder or create a new one. Assigning a group will allow you to filter and reset some options for all reminders within a group if desired. (Examples; Weather Reminders, Bills, Daily Items)

·        Notes  This is a free text field you can use to input notes to or provide a description for your reminder. (Example; Last auto insurance payment, 1/2/2015, it cost $756.14)

·        Mark Important  Use this flag to easily mark your important reminders. These can be quickly identified or filtered upon later.

·        Steps  For reminders that require multiple steps to perform you can itemize those in the steps screen. (more step details)

·        Trigger Type  Select the type of trigger associated with this reminder. Choose between Date/Time, Weather, Location, or To-do(no trigger). (more trigger details)

·        Notification  Select the notification options for this reminder. Choose from Pop-Up, Email, and/or text notification options. Note, email and text are purchasable add-ons. (more notification details)

Touch Save in the top right to save your reminder.

 

 

Details

            From the main screen you can view information about your reminder by touching it and access the reminder details screen. From this screen you can see any additional details about this reminder.

·        Name  Name of Reminder

·        Status  Active, Scheduled, Inactive

·        Group  The group with which this reminder is a member

·        Steps  A list of up to the 1st four steps for this reminder. Touching a step will launch the step interface screen for further functions.

·        Type  Specifies whether this reminder was created by you (User Created), or downloaded from the store (Downloaded)

·        Trigger  Displays the type of trigger associated with the reminder and its configuration details.

·        Notification  Shows the method of notification (if any) for this reminder

·        Notes  Displays the Notes provided about this reminder.

·        History Button/Link  Touching this will launch a log that shows when actions have been taken against this particular reminder.

The same actions provided on the main screen can also be taken here by using the options at the bottom. Done, Activate, Enable, Snooze, Edit, Disable, Delete

 

 

Triggers

To-Do  These reminders have no trigger associated with them. When creating the reminder just provide a name and save. The date and time this reminder was created is used to track due date status on the main screen.

 

Date/Time  Perhaps the most common type of trigger supply a date and time for this reminder to become due. There are also many repeat options available to assist you in any repeated items.

 

Weather  Unique to freeminders the weather trigger can be used to initiate your reminder based upon weather conditions. Many are available to choose from. Note that the weather conditions work in conjunction with the temperature, but not each other.

For example: If you create a reminder that has the following conditions;

Sleet, Snow, Below 40 degrees

This reminder will be triggered by any of the following conditions;

Sleet and below 40 degree temperature at some point that day

Snow and below 40 degree temperature at some point that day

Sleet & Snow & below 40 degree temperature at some point that day

This reminder will not be triggered by the following conditions;

Sleet & temperatures that do not drop below 40 degrees

Snow & temperatures that do not drop below 40 degrees

Below 40 degree temperature without sleet or snow

 

Location  Location reminders will notify you when you get within a provided proximity of a provided address. Perfect for those items when you need to be reminders to stop by the store, or post office, or dry cleaners, or

Notifications

Notifications are the methods with which you receive notice that your reminder is now active. freeminders offers 3 different methods to receive these alerts.

 

Pop-Up  The default notification type these are notifications that pop-up on your mobile device to notify you that your reminder is now Active. These notifications also create an entry in the Notifications area of your IOS device to view along side of other app notifications.

 

Email  This purchasable add-on will grant the ability to send emails notifications when a reminder is activated. You can configure up to 3 different email addresses and assign individual reminders to notify any or all of these. Another nice feature is that if you ever need to update an email address you can change the value in the settings screen and all of your already configured reminders will use the new email address to send notifications.

Steps

            The Steps screen is used for those reminders or items that take multiple steps. These will mainly be used with downloadable content but custom reminders are welcome to leverage steps as well. Recipes, workout routines, step by step guides are great example of reminders that can use these.

Create/Edit Screen  To access Screen for adding or editing steps touch the Steps tab at the top of the Add Reminder or Edit Reminder Screen

Create  To Create a new Step simple touch the +Add Step text field and start typing your Step. Touch the Add option to the right when you have completed typing in the Step.

Edit  To edit an existing step scroll to find and touch the step you would like to update. From here you can;

·        Update any and all text in the step

·        Move the Step Up in the Order

·        Move the Step Down in the Order

·        Delete the Step Entirely

Step Screen  The Steps screen is used to review and track which steps have been or still need to be completed. To access this screen go into the reminder details screen (tap the reminder) for a reminder with steps created (indicated by the steps icon on the main screen). From the reminder details screen simply touch any of the steps displayed there, this will take you to the Steps Screen

From this screen you can perform a number of things;

·        Mark steps complete by tapping them, mark them un-complete by tapping them again.

·        Mark all steps complete by tapping the select all box at the top, mark all steps un-complete by touching that select all box again.

·        Use a timer by using the interface built into the bottom of this screen.

·        Use a Stopwatch by using the same interface built into the bottom of this screen.

Other Notes and Definitions

Reminder Setting Hierarchy

Some settings used in a reminder, such as the notification method (email, text, etc.) and the location (physical address), can be configured/controlled at 3 different levels;

Reminder Level  These settings can be made directly on the reminder itself. The default setting for these will be what is currently configured as the default location in the application.

Group Level  Newly created groups or newly downloaded groups will have their location/notification settings default to what is configured as the application defaults. From the Group Details screen users should have the ability to change the default location. Upon make this change an OK/Cancel confirmation message ensuring the user would like to overwrite all of the current location/notification settings for reminders in that group should be displayed.

Application Level  Within the Settings menu there should be locations to set the default location & notification options. Downloaded or created groups as well as user created reminders should default to the location set here. Changes to this application default setting do not automatically update existing reminders through-out the application. A User would need to make updates at the group level for that.

 

 

The 3 Types of Downloaded Groups

          There are 3 different types of downloadable groups. The 3 types dictate primarily how reminders are activated.

 

Static next or recurring date. A date Based freeminder group is a group of which its reminders are tied to a specific date each year. For instance the Home Maintenance Reminders group would have reminders that state Clean the dryer vent pipes and check for leaks on the 1st Saturday in June every year.

When this type of group is downloaded and installed the application will schedule reminders to initially occur after the current date. For example if this group was installed on May 30th 2015 the 1st occurrence of the Clean the dryer Vents reminder would occur a week later on June 6th 2015. If the group was downloaded on June 7th 2015 this first occurrence of the Clean the dryer Vents reminder would not occur until next year until June 4th 2016.

Examples;

·       Every December 25th

·       The 1st Saturday in November

·       The last Sunday in January

·       The next Monday

·       2 Thursdays from today

·       The 1st of next month

·       The 2nd Tuesday of the month 2 months from now

 

Time after the group install/downloaded date. Another type of group would be one that can set reminders based upon the install date of the group. This type would often be used in conjunction with the ability to schedule reminders based upon the completion of other reminders.

An example of this would be a set of workouts that establishes a routine. The 1st workout session (Workout #1) will be Active at the time of installation. This initial reminder would need to have an Active date of the day the group was downloaded, if not completed that first day the reminder would become one day late, then 2 days, and so on until the reminder was completed.

There are going to be subsequent workout routines but those should not become active until the previous workout is complete. Workout #2 would need to monitor Workout #1 and then schedule its due date to by 2 days after Workout #1 is completed.

For example the 12 Week Beginner Weight Training group is downloaded and installed on the 1st of August. Upon installation Workout #1 is active and due to be completed that day. After 2 days of no action only the Workout #1 routine is Active and is now 2 days overdue. When that reminder is marked complete on the 3rd of August Workout #2 gets scheduled to be due 2 days later on the 5th of August. If Workout #2 is completed on the 5th Workout #3 becomes scheduled to be due on the 7th. These particular reminders are configured to become due 2 days after the parent reminder is completed, but this can be changed by reminder to be any amount of time (1 minute, 2 weeks, 1 year, instantly, etc.)

Examples:

·       The day the group is downloaded

·       2 Days After the group is downloaded

·       5 Days before the group is downloaded (this reminder would show up as being 5 days late)

·       2 Months after the group is downloaded

 

Time from a future date selected. Some reminders need to be scheduled based upon a fixed end goal date. Examples of these would be planning a wedding based upon a specified date, or having a baby given when the baby is due.

Using the Pregnancy reminder group as an example, upon download/installation of the reminder group a question would need to be asked of the user as to what is the due date of the Baby. When provided the reminders should be configured counting backwards from the due date. For example Discover the babys gender occurs 18 weeks before the due date. If a due date of August 1st 2015 is input then the reminder to Discover the babys Gender should be scheduled for March 28th 2015.

If the group is downloaded after such time that reminders would be scheduled then they are not activated unless those reminders are flagged as important, then they are Active and overdue by however many days they are late. Using the above example if the group was installed on March 31st 2015, and the August 1st 2015 date provided for the due date then the Discover the babys Gender is skipped and would never become Active because it is already 3 days past what that reminders due date would be. It would sit in a disabled state with the inactive status. If the Discover the Babys Gender were flagged as an important reminder by the author then that reminder would be Active upon installation and already 3 days past due.

Examples;

·       Target Date minus 14 days

·       The Saturday before the Target Date

·       3 Months before the Target Date

·       Target Date plus 5 days

·       2 Saturdays before the Target Date
